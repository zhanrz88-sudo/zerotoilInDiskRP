# ZeroToil Package Version Guide (for XAIOps / Worker Repo)

## Two Packages, One Module

| Package Name | Published By | Version Format | Use Case |
|---|---|---|---|
| `zerotoil` | Developer (local script) | `0.0.1.dev<YYMMDDHHMMSS>` | Dev / debug runs |
| `zerotoil-official` | Official CI pipeline | `0.0.0.N` (e.g., `0.0.0.1`, `0.0.0.2`, ...) | Production / stable runs |

Both packages provide `import zerotoil` — they are the same code, just
published under different names for access control.

---

## How to Identify Which Package to Install

The worker receives `InputParametersJson` with up to two relevant fields:

```json
{
  "ZEROTOIL_PACKAGE_VERSION": "0.0.0.3",
  "ZEROTOIL_USE_OFFICIAL": true
}
```

### Decision Logic

```python
version = params.get("ZEROTOIL_PACKAGE_VERSION")
use_official = params.get("ZEROTOIL_USE_OFFICIAL", False)
environment = os.getenv("CLOUD_SERVICE_NAME", "Prod")  # Prod, Test, Stage, ...

if not version:
    # No zerotoil needed for this job
    return

# Determine package name
if environment == "Prod" or use_official:
    package_name = "zerotoil-official"
else:
    package_name = "zerotoil"

# Install
FEED = "https://msazure.pkgs.visualstudio.com/One/_packaging/Storage-XI-feed/pypi/simple/"
subprocess.run([
    "pip", "install", "--no-deps",
    f"{package_name}=={version}",
    "--index-url", FEED,
], check=True)
```

### Summary Table

| Environment | `ZEROTOIL_USE_OFFICIAL` | Package Installed |
|---|---|---|
| **Prod** | _(ignored)_ | `zerotoil-official==<version>` |
| **Test / Stage** | `false` (default) | `zerotoil==<version>` |
| **Test / Stage** | `true` | `zerotoil-official==<version>` |

## Version Formats

### Dev versions (package: `zerotoil`)

```
0.0.1.dev260415033907
0.0.1.dev260428142312
```

- Generated by developers locally via `run_zerotoil_job.py`
- Each is a unique, disposable snapshot — not an ordered sequence
- Treat as an opaque build ID

### Official versions (package: `zerotoil-official`)

```
0.0.0.1
0.0.0.2
0.0.0.3
```

- Format: `0.0.0.N` where N is a simple incrementing integer
- Managed by the official CI pipeline
- Each number maps to a specific main-branch commit
- The latest version is tracked in:
  - **XScript-Templates repo**: `zero-toil/LATEST_OFFICIAL_VERSION`
  - **ADO feed**: [Storage-XI-feed](https://msazure.visualstudio.com/One/_artifacts/feed/Storage-XI-feed) → search `zerotoil-official`
  - **Pipeline build tags**: each build is tagged `vN` (e.g., `v0.0.0.3`)

### How to Find the Current Official Version

```bash
# Option 1: Read from the repo file
cat zero-toil/LATEST_OFFICIAL_VERSION | grep -v '^#' | grep -v '^$' | tail -1
# → 0.0.0.3

# Option 2: Check the ADO feed
pip index versions zerotoil-official \
  --index-url https://msazure.pkgs.visualstudio.com/One/_packaging/Storage-XI-feed/pypi/simple/

# Option 3: Check the pipeline build history
# Look for the latest build tagged "zerotoil-official" in ADO Pipelines
```

## Version Bump Commits

The official pipeline auto-commits to main after each successful build:

```
Bump zerotoil-official to version 0.0.0.3 [skip ci]

Package: zerotoil-official==0.0.0.3
Feed: Storage-XI-feed

Changes since last official build:
a1b2c3d Fix quorum loss detection logic
d4e5f6g Add escalation timeout parameter
```

These commits update `zero-toil/LATEST_OFFICIAL_VERSION` and include a
changelog of all `zerotoil/` code changes since the previous official build.

## Quick Reference: pip Install Commands

```bash
# Install a specific dev version
pip install zerotoil==0.0.1.dev260415033907 --no-deps \
  --index-url https://msazure.pkgs.visualstudio.com/One/_packaging/Storage-XI-feed/pypi/simple/

# Install a specific official version
pip install zerotoil-official==0.0.0.3 --no-deps \
  --index-url https://msazure.pkgs.visualstudio.com/One/_packaging/Storage-XI-feed/pypi/simple/

# Install the latest official version (no pin)
pip install zerotoil-official --no-deps \
  --index-url https://msazure.pkgs.visualstudio.com/One/_packaging/Storage-XI-feed/pypi/simple/
```
